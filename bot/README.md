# 📬 Telegram Mailing Bot

Telegram-бот для рассылки сообщений через пользовательские аккаунты Telegram. Оплата подписки через [CryptoBot](https://t.me/CryptoBot).

---

## 🚀 Возможности

### Рассылка
- Авторизация Telegram-аккаунтов через API ID / API Hash
- Рассылка сообщений от имени пользовательского аккаунта
- Персонализация: переменные `{username}` и `{first_name}` в тексте
- Настраиваемая задержка между сообщениями
- Перерывы: пауза на N секунд каждые M сообщений
- Прогресс-бар в реальном времени с кнопкой остановки
- Подробный отчёт после каждой рассылки
- Автоматическая обработка спам-блоков (через @SpamBot)
- Тестовое сообщение самому себе перед запуском

### Управление
- Шаблоны сообщений — сохранение и повторное использование текстов
- Списки получателей — сохранение групп юзернеймов
- Отложенные рассылки — планирование на конкретное время
- До 5 аккаунтов на пользователя (с покупкой доп. ячеек)

### Монетизация
- Подписочная модель через CryptoBot (@CryptoBot / @send)
- Free-тариф: 10 сообщений/день, задержка 5 мин, 1 аккаунт
- Pro-тариф: безлимит, настраиваемые параметры, до 5 аккаунтов
- Реферальная система с бонусными днями подписки

### Админ-панель
- Статистика продаж (сегодня / неделя / месяц / всего)
- Бан / разбан пользователей
- Управление подпиской (добавить/убрать дни, навсегда)
- Изменение цен (подписка, ячейки, реферальный бонус)
- Рассылка объявлений по всем пользователям бота

## ⚙️ Установка и запуск

### 1. Клонировать репозиторий
```bash
git clone <repo-url>
cd bot
```

### 2. Создать виртуальное окружение
```bash
python3 -m venv venv
source venv/bin/activate  # Linux/macOS
# или
venv\Scripts\activate     # Windows
```

### 3. Установить зависимости
```bash
pip install -r requirements.txt
```

### 4. Настроить конфигурацию

Откройте `config.py` и заполните:

```python
BOT_TOKEN = "ваш_токен_бота"
CRYPTO_PAY_TOKEN = "ваш_токен_cryptobot"
ADMIN_ID = 123456789  # ваш Telegram user_id
```

### 5. Запустить бота
```bash
python main.py
```

## 💡 Технологии

| Компонент | Технология |
|-----------|-----------|
| Бот (Bot API) | aiogram 3 |
| Клиент (MTProto) | Telethon |
| База данных | SQLite + aiosqlite |
| Платежи | CryptoPay API |
| Планировщик | APScheduler |

---

## 🔐 Безопасность

- Telethon-сессии хранятся как StringSession в SQLite
- Пароли 2FA **не сохраняются** (используются однократно)
- Забаненные пользователи полностью блокируются мидлварью
- Рекомендуется хранить `config.py` в `.gitignore` при публикации
